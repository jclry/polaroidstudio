# 需求文档 - 拟物化相机交互

## 介绍
将页面改造为拟物化风格，左侧为拍立得相机，右侧为照片展示区。体验“拍照-闪光-出片”的完整流程。

## 需求

### 需求 1 - 布局重构
**用户故事：** 页面应当分为左右两栏。
**验收标准：**
1.  **左侧相机区**：固定展示一张拍立得相机图片。
    -   *注意*：需要用户手动将提供的相机图片保存为 `assets/camera.png`。
2.  **右侧展示区**：用于放置生成好的拍立得照片。

### 需求 2 - 拟物化拍照交互
**用户故事：** 点击相机快门，模拟真实的拍摄体验。
**验收标准：**
1.  **快门热区**：在相机图片的“红点”位置设置点击区域。
2.  **触发上传**：点击快门时，触发系统的文件选择框（模拟拍摄/选图）。
3.  **闪光特效**：选中图片后，屏幕瞬间白屏闪烁（模拟闪光灯）。
4.  **出片动画**：
    -   **阶段一（打印）**：照片从相机底部出片口向下缓缓滑出。
    -   **阶段二（放置）**：照片从出片口位置平滑移动到右侧展示区的中心。
    -   **阶段三（定格）**：照片最终停留在右侧，带有轻微的固定旋转角度（如 3度）。

### 需求 3 - 现有功能保留
**用户故事：** 保留原有的颜色切换和下载功能。
**验收标准：**
1.  颜色选择器和下载按钮移动到右侧区域，位于照片下方。
2.  功能逻辑（换色、下载）保持不变。
