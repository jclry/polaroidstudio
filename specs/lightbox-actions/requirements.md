# 需求文档

## 介绍

调整 Lightbox 关闭动画的缓动曲线。用户反馈在时长一致的情况下，关闭动画感觉“突然变快”。这通常是因为 `ease-out` 类型曲线（`cubic-bezier(0.25, 0.8, 0.25, 1)`）在开始阶段变化较快。为了视觉上更均匀，关闭时应采用不同的缓动曲线，或者调整现有的曲线参数。

## 需求

### 需求 1 - 调整关闭动画曲线

**用户故事：** As a 用户, I want to 关闭动画平滑且速率均匀, so that 没有突然加速或减速的突兀感。

#### 验收标准

1. **When** 触发关闭 Lightbox 时, **the** 动画时长 **shall** 保持与打开时一致 (0.4s)。
2. **The** 缓动曲线 **shall** 调整为 `cubic-bezier(0.4, 0, 0.2, 1)` (更接近 ease-in-out) 或者标准的 `ease`，以避免 ease-out 在反向播放（缩小）时的“急起”感。
3. **Note**: 缩小动画如果使用强 ease-out，视觉上会感觉一开始缩得很快。改用更柔和的曲线可以缓解这种感觉。

## 技术细节

### 逻辑变更 (js/lightbox.js)

在 `closeLightbox` 中：
1.  设置 `card.style.transition = 'all 0.4s cubic-bezier(0.4, 0, 0.2, 1)'` (标准 Material Design deceleration curve，或类似)。
2.  或者尝试 `ease-in-out` (`0.42, 0, 0.58, 1`)。
3.  推荐：`cubic-bezier(0.4, 0, 0.2, 1)` 是一种非常通用的、自然的运动曲线。

### 为什么？
原曲线 `0.25, 0.8, 0.25, 1` 是强 ease-out。
- 打开时（变大）：快速变大，然后慢慢停下。很爽快。
- 关闭时（变小）：快速变小，然后慢慢停下。
- “快速变小”就是用户觉得“突然变快”的原因。
- 使用更接近线性的 ease-out 或 ease-in-out 可以让“变小”的过程没那么剧烈。
